LDAP vs Active Directory: the mental model

When I started working on identity labs, I confused "LDAP", "directory", "Active Directory" and "authentication". This post is my simplified mental model, based on coursework labs.

The short version

LDAP is a protocol to query directory information.
Active Directory is a Microsoft directory service that uses LDAP (plus many other components).

What problem do they solve?

  • Central identity: users, groups, and attributes live in one place.
  • Consistent logins: many machines can use the same identity source.
  • Auditability: you can reason about access and changes over time.

Practical checks I use in Linux labs

# Is the user visible via NSS?
getent passwd <username>

# Are groups visible?
getent group | head

# Is the client talking to the directory service?
systemctl status sssd --no-pager  # or nslcd/nscd depending on the lab

Placeholder: replace <username> and service names based on your lab (LDAP client vs AD join).

Common mistakes (and how I spot them)

  • Wrong base DN / domain name: look for errors in logs and config files.
  • DNS problems: domain joins and Kerberos are sensitive to DNS.
  • Time drift: Kerberos can fail if clocks are out of sync.
  • Permissions: user exists but cannot access home or resources.
