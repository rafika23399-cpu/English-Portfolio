How I verify a Linux service change

This is a short checklist I use after changing a Linux service configuration. The goal is simple: confirm the change is applied, the service is healthy, and the evidence is easy to show in a write-up.

1) Confirm what changed

Before restarting anything, I record the exact file I edited and keep a rollback copy.

Example commands
sudo cp /etc/<service>/<config> /etc/<service>/<config>.bak
sudo nano /etc/<service>/<config>

Placeholder: replace <service> and <config> with your real paths.

2) Restart + check status

sudo systemctl restart <service>
sudo systemctl status <service> --no-pager

If status is not "active (running)", I do not continue. I open logs first.

3) Read logs (fast)

sudo journalctl -u <service> -n 80 --no-pager
sudo journalctl -u <service> -S "10 min ago" --no-pager

Evidence tip: I screenshot the relevant error line (or success line) and include it in my PDF.

4) Validate network behavior (if applicable)

sudo ss -lntp | head
curl -I http://localhost:<port>  # if it's an HTTP service

If it is a remote service, I test from a second machine (client perspective).

5) Capture evidence for the portfolio

  • Before/after screenshot (or config diff)
  • systemctl status after restart
  • Relevant log lines (error or success)
  • Functional test output (curl/ssh/getent/etc.)
